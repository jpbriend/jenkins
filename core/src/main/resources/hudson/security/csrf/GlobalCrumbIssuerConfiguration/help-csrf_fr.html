Cross Site Request Forgery (ou CSRF/XSRF) est une attaque permettant à une tierce personne non autorisée d'exécuter des actions sur un site web en votre nom. Dans Jenkins, ceci peut permettre à quelqu'un d'effacer des projets, des builds ou de changer la configuration de Jenkins.

Lorsque cette option est activée, Jenkins vérifiera une valeur générée une seule fois, ou "crumb", sur toutes les requêtes effectuant des changements sur le serveur Jenkins. Cela inclut n'inporte quelle soumission de formulaire et les appels à l'API.

Activer cette option peut amener certain problèmes, tels que:

  • Certaines fonctionnalités de Jenkins (telle que l'API Remote) sont plus difficiles à utiliser lorsque cette option est activée.
  • Certaines fonctionnalités, notamment dans les plugins non testés avec cette option, peuvent ne plus fonctionner du tout.
  • Si vous accéder à Jenkins à travers un reverse proxy, il pourrait enlever l'en-tête HTTP CSRF, ce qui provoquera l'échec de certaines actions protégées.

Vous pouvez trouver plus d'information sur les failles CSRF ici.
